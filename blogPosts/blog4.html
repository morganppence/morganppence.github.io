In this lab, I learned how to use serial communication between Arduino and P5JS to create a game. Using my sensor box from Lab 3, I was able to use the two potentiometers allowing a signal from the Arduino to be sent to P5.

To start this lab, I worked on the Arduino to P5 serial setup. My two analog inputs from my sensor box, the potentiometers, now act as inputs for my screen interaction. With p5 up and running, I started working with my index.html, p5.serialport.js, sketch.js, and styles.css files.

I built a screen-based interface game in which the user’s goal is to get the circle from the top left corner of the screen past the obstacles and to the bottom right corner. By using the p5.serialcontrol app, I was able to sync my Arduino controller and my p5.js code. I created a canvas to match the screen width and height and added a few shapes to create the background of the game. I then created and mapped my ellipse (circle) to move on the x and y plane in coordination with my potentiometers.

I was able to use my Arduino code and circuit from the previous project, my sensor box. I did this by getting rid of the elememts relating to the LEDs and printing my potentiometer values.

I used the p5 website to generate elements of my interactive design. For example, I drews the shapes used as obsicles and the ellipse from the example pages on the p5 resource page:)
